# AI Support Report

## Purpose

AI tools were used only as a support resource for planning, refining documentation, and checking ideas. The execution of the application code, styling, and feature implementation was completed manually.

The main goal of using AI was to make the project clearer, not to generate finished code automatically.

## Tools Used and Use Cases

### ChatGPT

ChatGPT was used for:

- reviewing the Assignment 4 rubric and requirements
- verifying that the feature set aligned with the assignment goals
- refining wording in the README and technical documentation
- checking implementation ideas for clarity and correctness

### Editor-Based Suggestions

Editor suggestions were used for:

- small syntax and formatting cleanups
- consistent naming and indentation
- wording improvements in comments and documentation

These suggestions were reviewed carefully and only kept when they matched the project structure and made the code clearer.

## Benefits

Using these tools helped me work more efficiently by:

- clarifying the assignment requirements
- giving quick framing for documentation sections
- helping review the flow of feature implementation
- saving time on small formatting details

## Challenges and Limitations

I used AI in a limited way because:

- some suggestions were not aligned with the existing project structure
- some ideas were more complex than needed for this assignment
- the code still required manual correction and simplification
- I needed to test and verify every change independently

Because of this, AI was treated as a review and refinement assistant rather than a code generator.

## Learning Outcomes

AI support helped me learn more about:

- how to structure documentation clearly
- how to organize front-end logic in a readable way
- how to use browser storage features like `localStorage`
- how to make user-facing messages and validation more intuitive

## Responsible Use and Modifications

I used AI responsibly by:

- reviewing every suggestion before applying it
- modifying wording and code to fit my personal style and project needs
- simplifying or removing ideas that were not useful
- verifying functionality manually after each update

This kept the final project grounded in my own understanding and work.

## Final Reflection

AI was a support tool for planning and refinement, not a replacement for implementation. The final portfolio reflects my own coding, design, and testing work.
